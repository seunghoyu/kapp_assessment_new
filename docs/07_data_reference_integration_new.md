# kapp_assessment_new – 데이터 레퍼런스 통합 (new 반영)

> **취지**: origin의 05_data_reference_integration.md(KLI·KRIVET·조정 계수·벤치마크)를 전제로, **new 프로젝트에서** 데이터 레퍼런스를 어떻게 반영할지만 정리.  
> **목적**: new의 data/ 접근 방식·파일 제안·역할별 한 줄 가이드.

---

## 1. 전제 (origin 05 문서 기준)

- **KLI(한국노동연구원)**: 기업 규모 조정 계수·직종 분포·이직률·고령화 등.  
- **KRIVET(HCCP)**: KAPP 벤치마크·교육 시간·인당 교육비·스킬 갭·DX 역량 등.  
- **조정 계수·벤치마크**: 기업 규모·산업·직무별 가중치·평균/상위 백분위 등.

---

## 2. new에서 데이터 레퍼런스 반영 방향

### 2.1 현재 new
- **data/** 에서 정적 JSON·TS만 사용. API 없음.  
- **organizationData.json**, **industryBenchmarks.json**으로 부서·업계 스킬 벤치마킹만 수행.  
- 조정 계수·KLI/KRIVET 기반 벤치마크 수치는 아직 반영되지 않음.

### 2.2 new에 조정 계수·벤치마크용 JSON을 둘 경우

| 용도 | 파일명 제안 | 용도 한 줄 |
|------|-------------|------------|
| 기업 규모·산업별 조정 계수 | `data/adjustmentCoefficients.json` | (기업규모, 산업, 직종/역량) → 계수 값. KLI 기반. |
| 업계·직무별 벤치마크 수치 | `data/benchmarkWeights.json` 또는 기존 `industryBenchmarks.json` 확장 | 업계·직무별 평균/상위 10% 점수·교육 시간 등. KRIVET/HCCP 기반. |
| 역량–문항·직무 매핑 | `data/competencyQuestionMapping.json` | 역량 ID ↔ 문항·직무. O*NET/SQF/NCS 참고. |

- **역할**: 진단 결과 조정·결과지 “평균 대비” 문구·교육 추천 우선순위 등에 사용.  
- **연동 전**: 위 JSON을 data/에 두고, 프론트에서 읽어 “조정 점수”·“업계 평균 대비” 등 계산해도 됨.  
- **연동 후**: 백엔드가 동일 스키마로 API 제공하면, data/ 대신 API 응답으로 교체.

---

## 3. 역할별 new에서 데이터 접근 방법 (한 줄)

| 역할 | new에서 데이터를 어떻게 접근할지 |
|------|----------------------------------|
| **기획자** | data/ JSON 목록·문서(01~06)로 “어떤 화면이 어떤 데이터를 쓰는지” 확인. 조정 계수·벤치마크는 기획 문구·결과 해석에 반영할 필드만 정리. |
| **프론트엔드** | 필요한 화면에서 `@/data/` import. 조정/벤치마크는 서버에서 내려주는 값이 있으면 그대로 표시, 없으면 data/ JSON으로 폴백. |
| **백엔드** | 연동 시 조정 계수·벤치마크 테이블/캐시를 두고, 진단 제출·결과 조회·추천 API에서 일관 적용. new의 data/ 스키마를 API 응답 스키마 초안으로 사용. |
| **마케터** | “연구 기반·업계 벤치마크 반영” 메시징 시, data/에 둔 레퍼런스 출처(KLI·KRIVET)를 문서로만 참고. 숫자·공식은 기획·분석에 맡김. |
| **데이터 분석가** | data/ 내 JSON 스키마·역할 정리(05_project_brief_data_analyst_new.md). 조정 계수·벤치마크 산출 시 산출식·출처를 메타 문서로 남기고, new용 파일명(adjustmentCoefficients 등)으로 제공. |

---

## 4. 정리

- **new**에서는 당분간 **data/ 정적 JSON**만 사용.  
- **조정 계수·벤치마크**를 넣을 때는 위 **파일명·용도 제안**을 따르고, **역할별**로는 3절 표의 “new에서 어떻게 접근할지” 한 줄씩 적용하면 된다.  
- origin의 05_data_reference_integration.md와 이 문서를 함께 두면, “어떤 데이터를 왜 쓰는지”와 “new에서 데이터를 어떻게 접근·구현할지”를 맞춰 볼 수 있다.
