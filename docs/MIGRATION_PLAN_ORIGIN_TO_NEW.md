# origin → new 마이그레이션 계획

> **목적**: kapp_origin(HTML/CSS/JS) 기능을 kapp_assessment_new(Next.js App Router)로 순차 이관.  
> **우선순위**: 관리자 영역 먼저 → 이후 진단·학습자·교육·나의 성장·알림.

---

## 1. 개요

| 구분 | origin | new |
|------|--------|-----|
| **기술** | HTML, CSS, JS (멀티 페이지) | Next.js (App Router), React, Tailwind |
| **레이아웃** | 상단 네비, 스크롤 위주 | 사이드바 + 헤더 + 메인 |
| **데이터** | JS 인라인/목 데이터 | data/ 폴더 JSON·TS → 추후 API |
| **관리자** | admin.html 단일 페이지 내 섹션 | /dashboard, /dashboard/competency 등 라우트 분리 |

**이관 시 공통 원칙**
- 기존 data/ JSON이 있으면 활용. 없으면 origin 구조 참고해 JSON 추가 후 컴포넌트에서 import.
- HTML/CSS/JS 로직은 React 컴포넌트·훅·상태로 재작성. 스타일은 Tailwind로 통일.
- origin 참고 파일(admin.html, admin-*.js, admin-*.css 등)은 06_implementation_status_and_guide_new.md 6.4절 참고.

## 2. Phase 0: 대시보드 개요 UI 개선

### 2.1 변경·수정 내용

**현재**
- KPI 카드 4개 (총 직원 수, 진행 중 프로그램, 완료율, 평균 스킬 레벨)
- KPI/요약 영역이 카드 그리드로 분리되어 표시됨 (KPI 4개 + 요약/성과/액션)

**변경안 (채택)**
- **KPI 4개 + 요약/성과/액션**을 **단일 카드로 통합**하고, 카드 내부를 **테이블(정의형 리스트) 형태**로 표시.
  - **테이블 헤더**: 항목 / 값 / 상태·비고
  - **섹션 구분**: 핵심 KPI / 이번 달 요약 / 주요 성과 / 다음 액션
  - **표현**: 값(숫자·단위·Badge), 상태·비고(트렌드 아이콘·설명·상태 배지)

**효과**: 카드 그리드가 줄고, 개요 지표를 표 형태로 한 번에 스캔하기 쉬움.

### 2.2 대안 아이디어 (참고용)

- **A. KPI까지 한 카드에**: 상단에 KPI 4개를 작은 블록(2x2 또는 한 줄)으로 넣고, 아래에 이번 달 요약·주요 성과·다음 액션 리스트. → 한 카드에 모든 요약이 들어가지만, KPI 강조가 약해질 수 있음.
- **B. 다음 액션만 강조**: “상황 요약 & 다음 액션” 카드에서 “다음 액션”을 상단에 두고, 클릭 시 해당 페이지(인사이트·교육)로 이동하는 링크 부여. → 액션 중심으로 더 직관적.
- **C. 탭으로 구분**: 하나의 카드 안에 탭 3개(이번 달 / 성과 / 다음 액션). → 공간 절약하지만 한 화면에 다 안 보임.

**진행**: 2.1 변경안으로 구현 후, 필요 시 2.2 중에서 추가 반영.
