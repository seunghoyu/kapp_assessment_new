## 🎯 핵심 알고리즘

### 1. CAT (Computerized Adaptive Testing)
```javascript
// 난이도 자동 조정
if (userAnswer === correctAnswer) {
    // 정답 → 난이도 상승
    difficulty = difficulty === 'easy' ? 'medium' : 'hard';
    level = Math.min(5, level + 0.5);
} else {
    // 오답 → 난이도 하락
    difficulty = difficulty === 'hard' ? 'medium' : 'easy';
    level = Math.max(1, level - 0.5);
}
```

### 2. 문항 선택 로직
```javascript
// 1. Enhanced Questions 우선
if (window.enhancedQuestions) {
    questionPool = enhancedQuestions[category].filter(q =>
        q.industry === userData.industry &&
        q.job === userData.job &&
        !answeredQuestions.includes(q.id)
    );
    
    // 랜덤 선택
    question = questionPool[Math.floor(Math.random() * questionPool.length)];
}

// 2. Fallback to original
if (!question) {
    question = kappQuestionBank[category][difficulty][0];
}
```

## 📋 프로젝트 개요

### 🎓 KAPP 모델이란?

**K**nowledge (지식) · **A**pplication (적용) · **P**erformance (성과) · **P**roductivity (생산성)

4가지 차원에서 역량을 입체적으로 측정하여, 단순히 "무엇을 알고 있는가"를 넘어 **"실제로 성과를 낼 수 있는가"**를 검증합니다.

#### 1️⃣ Knowledge (지식) - 적응형 테스트(CAT)
- **기존 방식**: 모든 사용자에게 동일한 30개 문항 제공
- **KAPP 방식**: 정답률에 따라 AI가 난이도를 실시간 조정
  - 정답 → 더 어려운 문항 제시
  - 오답 → 더 쉬운 문항 제시
  - **최소 10~15개 문항으로 정확한 지식 경계면 측정**
- **효과**: 진단 시간 50% 단축 + 찍기 요소 배제

#### 2️⃣ Application (적용) - 디지털 인바스켓(E-tray)
- **기존 방식**: "이런 상황이라면 어떻게 하시겠습니까?" (주관적 선택)
- **KAPP 방식**: 가상 이메일함 + 메신저 환경 제공
  - 10분간 쏟아지는 5개 업무 요청
  - 우선순위 결정, 이메일 답장, 보고서 요약 **직접 수행**
  - 제한 시간 내 의사결정 프로세스 추적
- **효과**: "안다"는 주관성 배제 → **Show-how 수행 능력 검증**

#### 3️⃣ Performance (성과) - KPI 연동 시나리오
- **기존 방식**: "당신의 역량은 Lv.3입니다"
- **KAPP 방식**: 실제 KPI 개선 시나리오 제시
  - 예: "CAC 25,000원 유지하며 신규 고객 1만명 확보"
  - 현재 상황 데이터 제공 + 전략 선택
  - 각 선택지의 **예상 KPI 영향도** 수치화
- **효과**: 역량이 **비즈니스 성과로 전환되는 과정** 측정

#### 4️⃣ Productivity (생산성) - 업무 효율성 분석
- **기존 방식**: 정성적 업무 평가
- **KAPP 방식**: AI 업무 행동 패턴 분석
  - 5가지 업무의 **우선순위 결정** 테스트
  - 작업 순서, 소요 시간, 멀티태스킹 최적화 추적
  - **업무 효율성 지수(Efficiency Index)** 산출
- **효과**: **최소 리소스로 최대 아웃풋** 내는 업무 방식 검증

   - 5가지 업무 우선순위 결정

   - KAPP 대시보드로 자동 이동

## 📚 기술 문서

### Dreyfus 스킬 레벨 정의

| Level | 명칭 | 설명 | 특징 |
|-------|------|------|------|
| 1 | 초보자 (Novice) | 규칙을 엄격히 따름 | 상황 판단 없이 지침대로 수행 |
| 2 | 고급 초보자 (Advanced Beginner) | 제한적 상황 인식 | 유사 경험 활용 |
| 3 | 적임자 (Competent) | 계획 수립 가능 | 의도적 우선순위 설정 |
| 4 | 숙련자 (Proficient) | 직관적 상황 파악 | 패턴 인식 및 예측 |
| 5 | 전문가 (Expert) | 무의식적 역량 발휘 | 혁신적 문제 해결 |

### CAT 알고리즘 로직

```python
def adaptive_test(user):
    level = 3  # Start at medium
    questions_answered = 0
    
    while questions_answered < MIN_QUESTIONS:
        question = get_question(level, user.profile)
        answer = user.answer(question)
        
        if answer == question.correct:
            level = min(5, level + 0.5)  # Increase difficulty
        else:
            level = max(1, level - 0.5)  # Decrease difficulty
        
        questions_answered += 1
        
        if is_threshold_found(level, questions_answered):
            break
    
    return finalize_level(level)
```
