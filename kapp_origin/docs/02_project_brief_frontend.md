# 해커스 캠퍼스 Business NEXT GEN – 프론트엔드 관점 프로젝트 개요

> **역할**: 화면 구성, 데이터 소스, 상호작용, 상태·이벤트 로직  
> **초점**: 어떤 데이터를 어디서 쓰고, 사용자 액션 시 무엇이 일어나는지 (변수/클래스명은 제외)

---

## 1. 전체 페이지와 라우팅

| 경로 | 역할 | 진입 경로 |
|------|------|-----------|
| 홈 | 랜딩·소개 | 직접, 네비 링크 |
| KAPP 진단 | 정보 입력 + 진단 문항 | 홈·네비 "KAPP 진단" |
| 마이 대시보드 | 진단 결과 종합 표시 | 진단 완료 후 자동 이동, 네비 |
| 교육 큐레이션 | 추천·전체 과정·학습 경로 | 네비 |
| 나의 성장 | 1% 팁, AI 멘토, 인증서 | 네비 |
| 알림·설정 | 알림 on/off, 학습 설정 | 네비(또는 내부 링크) |
| 관리자 | 조직 통계·차트·고급 기능 | 네비 "관리자" |

- 페이지 간 이동은 **링크(앵커 포함)** 로 처리. SPA가 아니므로 화면 전환 시 전체 로드.
- 진단 완료 시 **저장 후 마이 대시보드 URL로 이동**하는 것이 유일한 프로그램적 리다이렉트.

---

## 2. 공통 레이아웃·요소

- **상단 네비게이션**: 브랜드명·서브타이틀, 홈·KAPP 진단·마이 대시보드·교육 큐레이션·나의 성장·관리자 링크. 현재 페이지에 따라 활성 표시.
- **푸터**: 링크·저작권 등(홈 등에서 사용).
- **공통 스타일**: 버튼(주요/보조/고스트), 카드, 폼 입력·셀렉트, 섹션 제목·설명. 아이콘은 아이콘 폰트 사용.

---

## 3. 데이터 소스와 저장

### 3.1 브라우저 저장소에 저장되는 것
- **진단 결과**: 제출 시 한 번 저장. 키 두 개에 동일 결과 저장(기존 대시보드·KAPP 대시보드 호환).
- **사용자 정보(홈 등)**: 선택 사항. 앱 전역 사용자 정보 저장용.
- **나의 성장**: 연속 학습 일수, 완료 팁 수, 마지막 완료 일자, 팁 이력.
- **알림 설정**: 푸시 허용 여부, 항목별 on/off.
- **학습 진행**: 알림 페이지에서 학습 진행 관련 설정.

### 3.2 프론트에서만 갖고 있는 데이터(목/더미)
- **진단**: 산업·직무 목록, 직급·연차·기업 규모 목록, 문항 뱅크(산업·카테고리·난이도별), E-tray·AI 워크플로우 시나리오·선택지·정답.
- **대시보드**: DNA 유형 정의, 샘플 결과(저장된 결과 없을 때), 직급 평균·시장 포지션·커리어 경로·연봉 예측용 목 데이터.
- **교육**: 과정 목록(제목·카테고리·난이도·기간·산업·추천 여부·커리큘럼·학습 결과 등), 역량별 학습 경로 단계·연결 과정.
- **관리자**: 부서별·기간별 점수, 임직원 테이블 행, 부서별 스킬 점수, 업계별 벤치마크 스킬 점수, 전략 키워드–스킬·역할·교육 매핑, 고성과자·리스크 목 데이터, ROI 계산 공식 입력값만 사용.

## 4. 페이지별 UI·로직

### 4.1 홈
- 히어로: 제목·부제·CTA 버튼(진단 시작·둘러보기).
- 통계/하이라이트: 숫자 카드(15분·97%·91% 등).
- 기능 소개: 카드 그리드, 앵커 링크.
- 클릭: 진단 페이지 이동 또는 앵커 스크롤. 스크롤 시 애니메이션(요소 등장) 적용 가능.

### 4.2 KAPP 진단
- **진행률 바**: 퍼센트·현재 단계 텍스트. 단계 전환 시 갱신.
- **1단계 – 정보 입력**  
  - 폼: 이름(필수), 이메일, 산업(필수, 선택 시 직무 목록 갱신), 직무(필수), 직급(필수), 연차(필수), 회사명, 기업 규모(필수), 진단 목표(다중 체크).  
  - 제출 시 입력값 검증 후 다음 단계로 전환, 입력값은 진단 상태에 보관.
- **2단계 – 문항**  
  - 문항 타입: 지식(선다형), 적용(시나리오), 성과(KPI·선다), 생산성(E-tray·AI 워크플로우).  
  - E-tray: 인바스켓 시뮬레이션, 타이머·선택 기록.  
  - AI 워크플로우: 시나리오·선택지·정답 채점.  
  - 적응형: 정오에 따라 난이도·다음 문항 변경.  
  - 다음/이전으로 진행, 마지막 제출 시 점수 집계·결과 객체 생성 → 저장 → 마이 대시보드로 이동.

### 4.3 마이 대시보드
- **로드 시**: 저장된 진단 결과 조회. 없으면 샘플 결과로 동일 UI 렌더.
- **나의 업무 DNA**: 점수·산업 등으로 DNA 유형 결정 → 아이콘·타입명·설명·트레이트 태그 표시. LinkedIn 공유·배지 다운로드 버튼(공유/다운로드 동작).
- **KAPP 점수**: 4개 점수 숫자·레벨·프로그레스 바. 레이더 차트: 나 vs 직급 평균 2개 데이터셋, 기존 차트 있으면 제거 후 재생성.
- **시장 포지션**: 현재 포지션·백분위·점수·강점/개선/평균 스킬 수·시장 수요도·성장 기회 타임라인. 산업군 벤치마크 레이더(상위 10% vs 나). 추천 액션 리스트. “맞춤 성장 플랜” 버튼 → 교육 등 페이지로 이동 가능.
- **커리어 경로**: 산업·직무 기반 타임라인·학습 경로 블록. “추천 강의 둘러보기” → 교육 페이지.
- **시장 벤치마킹**: 막대 차트 + 동일 직급 평균·나·상위 백분위 텍스트.
- **스킬 성장 추이**: 6개월 시계열 라인 차트.
- **AI 인사이트**: 점수·갭 기반 카드 여러 개.
- **IDP**: 스킬 갭 기반 목표·기간·추천 교육 자동 생성.
- **게이미피케이션**: 레벨·XP·배지·연속 학습 등, 저장된 데이터 또는 진단 결과 기반 계산.
- **연봉 예측**: 3년 시계열. 산업·직무 기반 목 데이터.
- **인쇄/다운로드**: 브라우저 인쇄 대화상자. 재진단 버튼 → 진단 페이지.

### 4.4 교육 큐레이션
- **로드 시**: 과정 목록 로드. 저장된 진단 결과 있으면 맞춤 추천 과정 필터·추천 사유 문구 설정.
- **필터**: 산업군·역량 분야·난이도·학습 기간 셀렉트. 변경 시 목록 필터링만 수행(서버 요청 없음).
- **맞춤 추천**: 추천 과정 카드 그리드. 클릭 시 상세 모달.
- **전체 과정**: 그리드/리스트 뷰 전환 버튼. 카드 클릭 시 상세 모달.
- **상세 모달**: 제목·카테고리·난이도·기간·설명·커리큘럼·학습 결과·수강 신청(또는 외부 링크). 닫기 버튼으로 모달 숨김.
- **추천 학습 경로**: 역량별 단계 블록, 각 단계에 연결된 과정 표시.

### 4.5 나의 성장
- **오늘의 1% 팁**: 카드(카테고리·제목·내용). “새 팁 받기” → 랜덤 팁으로 교체. “적용 완료” → 완료 수·연속 일수·주간 진행 갱신 후 저장. “나중에 보기” → 이력에 추가 등.
- **연속 학습·완료 팁·주간 목표**: 저장된 값 표시.
- **팁 이력**: 이전 팁 목록, 클릭 시 해당 팁 다시 표시 가능.
- **AI 멘토링**: 커리어 목표 카드 선택 → 채팅 영역 활성화. 사용자 메시지 입력 → 시나리오별 답변(교육 추천·기간·전략·연봉·동기부여 등) 생성하여 표시.
- **역량 인증서**: 진단 결과 기반 미리보기. LinkedIn 공유·다운로드.

### 4.6 알림·설정
- 푸시 알림 권한 요청 버튼 → 권한 요청 후 허용 시 저장.
- 항목별 토글(오늘의 팁·연속 학습 알림·학습 경로 추천 등) → 변경 시 설정 저장.
- 학습 진행 관련 설정 로드·저장.

### 4.7 관리자
- **요약 카드**: 고정 숫자 또는 목 데이터(임직원·진단 완료·교육 이수·평균 점수).
- **부서별 역량 차트**: 기간(월) 셀렉트 변경 시 해당 월 부서별 점수로 막대 차트 다시 그림. 기존 차트 인스턴스 제거 후 재생성.
- **역량 분포 차트**: 파이/도넛, 한 번 생성 또는 데이터 변경 시 갱신.
- **진행 현황 카드**: 필터(전체/진행중/완료) 클릭 시 카드 목록 필터링(표시/숨김).
- **임직원 테이블**: 목 데이터 행 렌더. 검색 입력 시 클라이언트 필터. “임직원 추가” → 안내 또는 모달.
- **스킬 갭 히트맵**: 부서·스킬 카테고리 선택 후 “새로고침” → 부서별 스킬 셀 색상(점수 구간별)·교육 배정 버튼. 버튼 클릭 시 안내 메시지.
- **업계 벤치마킹 레이더**: 비교 부서·비교 업계 셀렉트 선택 후 “비교 분석” 또는 선택 변경 시 바로 레이더 갱신 + 인사이트 영역(개선 필요·경쟁 우위) 재계산·렌더. 개선 계획 수립 버튼 → 안내.
- **전략–스킬 매핑**: 전략 키워드 입력 후 “AI 분석” → 키워드 매칭으로 추천 인력·교육·로드맵 표시. 인력 배정·일정 등록 버튼 → 안내.
- **ROI 계산기**: 인원·1인당 비용·스킬 향상 % 입력 후 “계산” → 총 투자·예상 수익·순이익·ROI % 표시, 상세 분석 블록·보고서 다운로드 버튼(안내 또는 파일 생성).
- **고성과자 분석**: 팀 선택 후 “분석” → Golden Standard 레이더·갭 분석 블록 갱신.
- **스킬 리스크**: 리스크 레벨 셀렉트·재분석 버튼 → 긴급/높음/보통/낮음 카운트·상세 리스트 갱신.

## 5. 차트·시각화 로직

- **레이더**: 2개 데이터셋(예: 나 vs 비교대상). 축 라벨은 첫 번째 데이터셋 기준. 스케일 0–100. 반응형, 컨테이너 크기 변경 시 리사이즈. 재렌더 전 기존 인스턴스 제거.
- **막대·라인·파이**: 데이터 배열·라벨로 한 번에 그리기. 기간/필터 변경 시 데이터만 바꿔서 다시 그리거나 인스턴스 업데이트.
- **히트맵**: 그리드 셀, 점수 구간별 배경색(예: 0–40 긴급, 41–60 주의, …). 클릭/버튼은 셀 또는 행 단위 액션(교육 배정 등).

---

## 6. 폼·유효성

- 진단 정보 입력: 필수 필드 비어 있으면 제출 차단·메시지.
- 관리자 ROI: 숫자 입력만 허용, 범위 제한(예: 1–100).
- 기타 폼: 필요 시 빈 값·형식 검사 후 제출 또는 버튼 활성화.

---

## 7. 모달·토스트

- **교육 과정 상세**: 오버레이 + 모달 본문. 닫기(X)·오버레이 클릭 시 닫기. 본문 스크롤 가능.
- **안내 메시지**: 버튼 클릭 시 브라우저 알림 또는 토스트로 “완료되었습니다” 등 표시. 모달로 상세 안내할 수도 있음.

---

## 8. 반응형·접근성

- 주요 영역: 그리드·카드가 좁은 화면에서 세로 쌓이거나 탭/아코디언으로 전환.
- 차트: 컨테이너 비율 유지 또는 최소 높이로 스크롤.
- 버튼·링크·폼 컨트롤: 포커스·키보드 동작 고려. 아이콘만 있으면 텍스트 또는 aria 라벨 보완.

---

## 9. 정리

- **상태**: 진단 단계·필터 선택·모달 열림 등은 메모리 상태. 새로고침 시 저장된 것만 복원(진단 결과·성장·알림 설정).
- **외부 의존**: 차트 라이브러리·아이콘 폰트·폰트 로드. API 호출 없음.
- **확장**: 산업·직무·문항·과정·전략 키워드·부서·업계 데이터만 추가하면 동일 로직으로 확장 가능.
