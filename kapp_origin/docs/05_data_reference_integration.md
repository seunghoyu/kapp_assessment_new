# AI 활용 역량 진단 서비스 – 데이터 레퍼런스 통합 정리

> **취지**: 역량 진단 서비스 구축에 필요한 자료·링크를 체계적으로 모아, 정합성 검토 및 기획 단계에서 빠르게 재확인·활용하기 위함.  
> **목적**: 근거가 되는 데이터 소실 방지, 역할별(데이터 분석·기획·프론트·백엔드·마케팅) 데이터 접근 가이드 제공.

---

## 1. 데이터 레퍼런스 요약

### 1.1 한국노동연구원(KLI) – 사업체패널조사 기반 고용현황

| 항목 | 내용 |
|------|------|
| **출처** | 한국노동연구원, 2023년 발간 보고서 |
| **링크** | https://www.kli.re.kr/kli/newestPblctList.es?sch_yr=&sch_type=&sch_keyword=&sch_newest=&mid=a10101000000&nPage=3 |
| **활용 목적** | 기업 규모 조정 계수 산출, 기업 규모·산업 페르소나 정교화 |

**핵심 데이터 요약**  
- **직종 분포**: 대기업 전문·사무직 37.8~40.6%, 서비스·판매직 비중 높음. 중소기업은 생산·단순직 비중이 대기업(21.7%)의 약 2배(39.9%).  
- **노사 관계**: 대기업 유노조 사업체 33.7% vs 중소기업 12.9%.  
- **고령화**: 중소기업 55세 이상 26.6% vs 대기업 17.4%, 고령화 속도도 중소기업이 더 빠름.  
- **이직률**: 전체 이직률 중소기업 23.0% vs 대기업 18.7%. 자발적 이직에서 차이 큼.  
- **고용 형태**: 대기업은 간접고용·비정규직 비중 상대적으로 높음.

**서비스 반영 방향**  
- 기업 규모 선택 시 **문항·시나리오·가중치 차별화**:  
  - **중소기업**: 공정 효율화·현장 실행력·멀티태스킹·세대 간 소통·숙련 기술 전수·인재 리텐션 강화.  
  - **대기업**: 복잡한 의사결정·고객경험(CX)·부서 간 협업·노사 갈등 관리 강화.  
- **조정 계수**: 직종 분포 → 역량 중요도 가중치, 이직률 → 성과(P) 벤치마크, 고령화 → 생산성(P) 기대치 보정, 노조/고용형태 → 적용(A) 난이도 계수.  
- 계수 종류: 직종 복잡도 계수, 조직관리 난이도 계수, 환경 유연성 계수, 인력 숙련 구조 계수 등 → **가중치 계산 후 조정 계수**로 사용.

### 1.2 한국직업능력연구원(KRIVET) – 인적자본기업패널조사(HCCP)

| 항목 | 내용 |
|------|------|
| **출처** | 한국직업능력연구원(KRIVET) 자료실 |
| **링크** | https://www.krivet.re.kr/kor/sub.do?menuSn=12 (인적자본기업패널조사 HCCP 2025 등) |
| **활용 목적** | KAPP 정량적 벤치마크, 기업 규모·산업별 조정계수 근거, AI·DX 역량 진단 항목·루브릭 설계 |

**핵심 데이터 요약**  
- 기업의 **교육훈련비·교육 시간·성과** 데이터.  
- 업종별/직무별 **평균 교육 시간·인당 교육비**.  
- 인적자본 투자와 **재무 성과(매출·영업이익)** 상관관계.  
- 기업 규모별 **교육 격차·디지털 전환 수준**.  
- 산업별 **스킬 갭(Skill Gap)**.  
- **디지털 도구 활용·혁신 활동** 관련 항목.  
- **향후 강화 계획 역량**, 온/오프라인·마이크로러닝 선호도 등.

**서비스 반영 방향**  
- **지식(K)·성과(P) 기준점**: 업종/직무별 평균 교육 시간·인당 교육비를 결과지 “평균치”로 표시. (예: 사용자 지식 습득 노력이 해당 산업 평균 대비 어느 수준인지.)  
- **비즈니스 임팩트 예측**: “역량 10% 상승 시 조직 생산성 기여 약 X%” 등 HCCP 상관관계 기반 로직.  
- **기업 규모·산업별 조정계수**: 대·중소 격차, 산업별 스킬 갭 반영 → 적용(A)·생산성(P) 가중치·배점 보정.  
- **AI 워크플로우·DX**: HCCP의 디지털 도구 활용·직무 수행 방식 변화 데이터 → 생산성(P) 평가 루브릭·디지털 인바스켓 시나리오 설계.  
- **교육 큐레이션**: “기업이 향후 강화 계획 중인 역량” → 해당 역량 부족 사용자에게 관련 강의 우선 추천. 학습 형태(온/오프·마이크로러닝) 데이터로 추천 형태 제안.

## 2. 조정 계수·벤치마크 활용 체계(요약)

- **직종 분포** → 대기업: 사무/전문직 역량 가중치 상향(예: 1.2). 중소기업: 현장 실행력·멀티태스킹 가중치 상향, 사무직 문항 비중·난이도 조정.  
- **이직률** → 기업 규모별 기본 이직률을 상수로 두고, 리더의 팀 이직률 관리 성과를 성과(P) 벤치마크·가산에 반영.  
- **고령화** → 중소기업·고령화 높은 환경에서 AI 도입 성과 시 생산성(P) 계수 상향(디지털 전환 난이도 반영).  
- **노조/고용형태** → 대기업: 노사·다양한 고용형태 갈등 시나리오, 적용(A) 난이도 계수 적용. 중소기업: 자원 부족·빠른 실행 중심 시나리오, 평가 기준 분리.  
- **HCCP 교육·성과** → 업종/직무별 평균 교육 시간·비용을 결과지 벤치마크·비즈니스 임팩트 문구에 사용.

## 3. 역할별 데이터 접근 가이드

### 3.1 데이터 분석가

**역할**  
- 레퍼런스 데이터 수집·정제·정합성 검토.  
- 조정 계수·벤치마크 수치 산출 및 검증.  
- 기획·백엔드에 “쓸 수 있는 형태”로 데이터 정의·제공.

**접근 방법**  
1. **원천 데이터 관리**  
   - KLI·KRIVET(HCCP)·잡코리아·사람인·O*NET·SQF·NCS 등 **출처·버전·수집일** 명시한 데이터 딕셔너리 및 raw/정제본 경로 유지.  
   - 기업 규모·산업·직종·직무 코드 체계를 **서비스 코드와 1:1 매핑 테이블**로 정리(예: KLI 직종 코드 ↔ 내부 job_category_id).

2. **조정 계수 산출**  
   - 직종 복잡도·조직관리 난이도·환경 유연성·인력 숙련 구조 등 **차원별 지표 정의** 후, KLI·HCCP 수치로 계수 산출.  
   - 산출식·가정·데이터 출처를 **계수 메타 문서**로 남겨, 기획·정합성 검토 시 재현 가능하게 유지.

3. **벤치마크 테이블 설계**  
   - **산업×직무×(기업규모)** 조합별: 평균 교육 시간·인당 교육비·이직률 기준값·고령화 지수 등.  
   - KAPP 차원별 “업계 평균” “상위 10%” 등 **백분위/구간** 정의 후, HCCP·내부 집계로 수치 채우기.

4. **정합성 검토**  
   - 진단 문항–역량–O*NET/SQF/NCS 매핑이 **일관된지** 주기적으로 점검.  
   - 조정 계수 적용 전후 **점수 분포·편향** 검토(예: 기업 규모별 평균 점수 차이가 지나치지 않은지).

5. **교육 큐레이션 연계**  
   - HCCP “향후 강화 계획 역량”·학습 형태 선호를 **역량 ID–강의 ID–우선순위·형태** 매핑 테이블로 정리.  
   - 데이터 분석가가 이 테이블의 초기 버전을 설계하고, 백엔드가 API/DB로 서빙.

**산출물 예시**  
- 데이터 딕셔너리(엑셀/Notion/Confluence).  
- 조정 계수 메타 문서 + 계수 CSV/DB 스키마.  
- 산업·직무·기업규모별 벤치마크 테이블(버전 관리).  
- 문항–역량–직무 매핑 검증 리포트.

### 3.2 기획자

**역할**  
- “무엇을 측정하고, 어떤 사용자에게 어떤 경험을 줄지” 정의.  
- 데이터 레퍼런스를 **페르소나·시나리오·결과 해석**에 연결.

**접근 방법**  
1. **페르소나 정교화**  
   - KLI 기업 규모별 직종·노사·고령화·이직률 차이를 **페르소나 카드**에 반영.  
   - 예: “중소기업 생산직 관리자” → 공정 효율화·현장 실행·세대 간 소통·리텐션 시나리오 강조. “대기업 사무직 팀장” → 의사결정 복잡도·CX·부서 협업·노사 시나리오 강조.

2. **문항·시나리오 기획**  
   - 인바스켓·AI 워크플로우 문항의 **상황 설정**을 직종·기업 규모에 따라 분기.  
   - 데이터 분석가가 정리한 “역량–문항–기업규모/산업” 매핑을 기반으로, **어떤 필터 조합에 어떤 문항 세트가 나갈지** 시나리오 플로우 정의.

3. **결과 해석·메시징**  
   - HCCP 기반 “평균 대비” 문구·비즈니스 임팩트 문장을 **기획 문구**로 정의.  
   - 조정 계수 적용 결과(예: 중소기업 리더 이직률 관리 가산)가 결과지·대시보드에 **어떤 블록·문구로 노출될지** 기획.

4. **교육 큐레이션 기획**  
   - “기업이 강화 계획 중인 역량” 데이터를 **추천 우선순위·추천 사유 문구**에 반영하는 룰을 기획.  
   - 온/오프·마이크로러닝 등 **학습 형태 추천** 노출 위치·조건 정의.

**데이터 활용 포인트**  
- 레퍼런스 요약본·페르소나별 시나리오 매트릭스를 자주 참고.  
- 조정 계수·벤치마크가 “기획서 상의 사용자 가치”와 맞는지 정합성 검토 시 데이터 분석가와 함께 점검.

### 3.3 프론트엔드

**역할**  
- 진단 입력(기업 규모·산업·직무·직급·경력 등) 옵션 표시.  
- 조정 계수·벤치마크가 **반영된 결과**를 대시보드·결과지에 시각화·텍스트로 노출.  
- 데이터는 “어디서 오는지(API/상태)”만 알고, 계수 산출 로직 자체는 백엔드에 맡김.

**접근 방법**  
1. **마스터 데이터 소비**  
   - 산업·직무·직급·경력·기업 규모 **옵션 목록**은 API에서 받아 옵션 값·라벨만 표시.  
   - O*NET→국내·SQF·NCS 매핑 결과는 “역량 라벨·문항 카테고리” 수준으로 API 응답에 포함되면 그대로 표시.

2. **진단 플로우**  
   - 사용자가 선택한 **기업 규모·산업·직무**를 문항 요청/제출 시 함께 전달.  
   - 서버가 해당 조합에 맞는 문항·시나리오·가중치를 적용해 반환하므로, 프론트는 **문항 ID·지문·선택지·진행 순서**만 렌더링.

3. **결과·대시보드**  
   - **원점수·조정 점수·벤치마크 대비 문구**를 API에서 내려주면, 그대로 숫자·퍼센트·레이블로 표시.  
   - “업계 평균 대비”, “역량 10% 상승 시 조직 기여 X%” 등 **고정 문구 템플릿**은 기획 문구에 맞춰 프론트에 두거나, API에서 문구 키/텍스트를 받아 표시.

4. **교육 추천**  
   - 추천 과정 ID·사유·우선순위·학습 형태를 API에서 받아 카드·필터에 반영.  
   - 데이터 출처(HCCP 기반 등)는 “추천 근거” 문구로만 노출하면 됨.

**데이터 활용 포인트**  
- “기업 규모/산업/직무가 바뀌면 문항·가중치가 달라진다”는 **플로우**만 이해.  
- 실제 계수·벤치마크 숫자는 백엔드/분석가가 관리하고, 프론트는 **응답 스키마·필드 의미**만 맞추면 됨.

### 3.4 백엔드

**역할**  
- 조정 계수·벤치마크·문항–역량 매핑을 **서버에서 일관되게 적용**.  
- 진단 제출 → 점수 집계·조정·저장 → 결과·대시보드·추천 API로 전달하는 **로직과 데이터 흐름** 담당.

**접근 방법**

**1) 데이터가 서버에 “닿는” 구조**  
- **마스터·레퍼런스 데이터**: DB 또는 설정 저장소에 보관.  
  - **코드/옵션**: 산업, 직무, 직급, 경력, 기업 규모 (잡코리아·NCS 등과 매핑된 코드).  
  - **조정 계수**: (기업규모, 산업, 직종/역량 등) → 계수 값. 데이터 분석가가 산출한 CSV/테이블을 주기 반영.  
  - **벤치마크**: (산업, 직무, 기업규모) → 평균 교육시간·인당 교육비·이직률 기준·KAPP 차원별 평균/백분위. HCCP·KLI 기반 테이블.  
  - **문항 뱅크**: 문항 ID, 산업, K/A/P/P, 난이도, 유형, 지문, 선택지, 정답, (선택) 기업규모별 시나리오 분기.  
  - **역량–문항–직무 매핑**: O*NET·SQF·NCS 기반 역량 ID ↔ 문항 ID ↔ 직무/산업.

- **진단 세션·결과**:  
  - 세션: userId, 시작/종료 시각, 선택된 기업규모·산업·직무·직급·경력.  
  - 응답: 문항 ID별 선택/정오.  
  - 결과: 원점수(K,A,P,P), **조정 계수 적용 후 점수**, 종합 점수, DNA 유형, (선택) 문항별 로그.

**2) 로직 구성**  
- **문항 노출**  
  - 진단 시작/다음 문항 요청 시: (기업규모, 산업, 직무, 직급, 경력) + 이전 응답(적응형일 경우) 전달.  
  - 서버: 조정 계수·역량 가중치를 적용한 “이번에 낼 문항 세트” 결정. 기업 규모별 시나리오 분기(인바스켓·AI 워크플로우) 선택.  
  - 응답: 문항 ID, 지문, 선택지, 유형, (선택) 제한 시간.

- **점수 집계·조정**  
  - 제출 시: 문항별 정오·소요 시간 등으로 **원점수** 계산(K,A,P,P 차원별).  
  - **조정**: (기업규모, 산업, 직무 등)에 해당하는 조정 계수 테이블 조회 → 곱/가산 적용 → **조정 점수** 저장.  
  - 이직률 벤치마크·고령화 보정 등은 “성과(P)” “생산성(P)” 차원에만 선택 적용 가능하도록 플래그/규칙으로 두면 됨.

- **결과·대시보드 API**  
  - 결과 조회: 저장된 조정 점수·종합·DNA 반환.  
  - 벤치마크 대비: (산업, 직무, 기업규모)로 벤치마크 테이블 조회 → “평균 대비 N%”, “상위 M%” 등 텍스트/수치 생성해 응답.  
  - 비즈니스 임팩트: HCCP 기반 공식(역량 상승 ↔ 생산성 기여)으로 예측 문구 생성(또는 미리 계산된 문구 키 반환).

- **교육 추천**  
  - 진단 결과(부족 역량·산업·직무) + “기업 강화 계획 역량” 매핑 테이블 + 과정–역량 매핑 → 추천 과정 ID·우선순위·사유·학습 형태 반환.  
  - 학습 형태는 HCCP 선호도 데이터 기반 규칙 또는 사용자/조직 설정으로 선택.

**3) API 설계 요약**  
- `GET /api/v1/master/industries`, `/jobs`, `/company-sizes`, `/positions`, `/experience-levels`  
  - 옵션 목록(코드, 라벨, 정렬).  
- `POST /api/v1/assessment/sessions`  
  - 세션 생성. body: userInfo(기업규모, 산업, 직무, 직급, 경력 등).  
- `GET /api/v1/assessment/sessions/:id/next-questions`  
  - 다음 문항. query/body: 이전 응답(적응형). 서버가 문항·시나리오 분기 결정.  
- `POST /api/v1/assessment/sessions/:id/submit`  
  - 응답 제출. body: 문항별 응답. 서버: 원점수 → 조정 → 저장 → 결과 반환.  
- `GET /api/v1/assessment/results/:id`  
  - 결과 상세(조정 점수, DNA, 벤치마크 대비 문구, 비즈니스 임팩트).  
- `GET /api/v1/courses/recommendations`  
  - query: resultId 또는 userId. 응답: 과정 ID, 사유, 우선순위, 학습 형태 등.  
- 관리자·벤치마크·조정 계수 **관리 API**는 내부/어드민 전용으로 설계(CRUD·버전·배포).

**4) 보안·성능**  
- 조정 계수·벤치마크·문항 뱅크는 **캐시(Redis 등)** 로 읽기 성능 확보.  
- 계수·벤치마크 갱신 시 캐시 무효화·버전 키로 일관성 유지.  
- 진단 결과·개인 식별 정보는 권한·암호화·감사 로그 적용.

### 3.5 마케터

**역할**  
- “데이터 기반 진단” “업계 벤치마크” “기업 규모·산업 맞춤”을 **메시지·사례·지표**로 활용.  
- 데이터는 직접 다루지 않고, **기획·분석가가 정리한 “무엇이 다른지”**를 언어화.

**접근 방법**  
1. **메시징**  
   - KLI·HCCP 등 **공인된 연구 기반**임을 강조. (예: “한국노동연구원·한국직업능력연구원 데이터 기반 맞춤 진단”.)  
   - 기업 규모·산업별 **다른 문항·다른 기준**을 “페르소나 맞춤” “공정한 비교”로 전달.  
   - HCCP 연계 “역량 상승 시 조직 기여” 문구를 **ROI·성과 메시지**로 활용.

2. **콘텐츠·사례**  
   - “대기업 팀장 vs 중소기업 현장 관리자, 진단이 어떻게 다른가”를 **비교 인포그래픽·블로그**로 제작.  
   - 조정 계수로 “같은 점수라도 맥락이 반영된다”는 점을 **공정성·신뢰도** 스토리로 사용.

3. **지표**  
   - 진단 완료율·대시보드 이탈률·교육 추천 클릭률 등 **행동 지표**와 함께, “기업 규모별·산업별 완료율/만족도”를 집계해 **타깃별 성과**로 보고.  
   - 데이터 분석가가 만든 **벤치마크·계수 메타 문서**를 “검증 가능한 방법론”으로 간단히 요약해 제안서·백서에 첨부.

**데이터 활용 포인트**  
- 숫자·공식은 분석가·기획에 맡기고, **해석·스토리·메시지**에 집중.  
- 레퍼런스 링크·기관명을 **출처 표기**해 신뢰도를 높임.

## 4. 정합성 검토·아카이브 활용

- **보드/댓글 아카이브**: 위 레퍼런스 링크·요약·“어디에 쓸지”를 한곳(Notion·Confluence·Google 시트 등)에 모아 두고, 댓글로 추가 자료·버전 갱신을 계속 적재.  
- **정합성 검토 시**:  
  - 문항–역량–직무 매핑이 O*NET·SQF·NCS와 맞는지.  
  - 조정 계수 적용 전후 점수 분포·편향.  
  - 벤치마크 수치와 결과지 문구·대시보드 수치 일치.  
  - 데이터 분석가가 계수·벤치마크 메타 문서를 기준으로 검증하고, 기획·백엔드가 기획서·API 스펙과 대조.

- **데이터 소실 방지**:  
  - 원천 URL·보고서명·수집일·버전을 **데이터 딕셔너리·레퍼런스 목록**에 필수 필드로 두고, 분석가가 주기적으로 점검.  
  - 조정 계수·벤치마크 테이블은 **버전·이력** 관리해 “언제 어떤 근거로 적용했는지” 추적 가능하게 유지.
