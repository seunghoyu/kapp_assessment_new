# kapp_assessment_new – 기획자 관점 프로젝트 개요

> **역할**: 제품/서비스 기획, 사용자 여정, 기능 범위, 데이터 흐름  
> **초점**: 무엇을, 누구를 위해, 어떤 순서로 제공하는지  
> **기준**: kapp_origin(HTML/CSS/JS) 대비 **Next.js 기반 new** 프로젝트 전용 정리

---

## 1. 프로젝트 비전과 목표

### 1.1 한 줄 정리
**AI 기반 KAPP 역량 진단으로 임직원 성장을 측정·증명하고, 진단 결과에 맞춘 교육 큐레이션과 관리자용 조직 역량 인사이트를 제공하는 원스톱 HR 솔루션** — new에서는 **Next.js 기반**, **사이드바·헤더 레이아웃**, **데이터 표시 간소화**, **data/ JSON 더미 데이터**로 단계적 구축 중.

### 1.2 핵심 가치 (origin과 동일, new에서의 구현 범위만 다름)
- **진단**: KAPP 4차원(Knowledge, Application, Performance, Productivity) 측정 — **new 미구현**
- **성장 증명**: 마이 대시보드(DNA·점수·레이더·시장 포지션 등) — **new 미구현**
- **교육 연계**: 진단 기반 맞춤 추천·학습 경로·IDP — **new 미구현**
- **조직 인사이트**: 관리자 대시보드(요약 KPI·부서 차트·역량/벤치마크·진행 현황·임직원·인사이트·히트맵·업계 벤치마킹·전략 매핑·ROI·고성과자·스킬 리스크) — **new에는 일부만 구현**

### 1.3 사용자 역할
| 역할 | 대상 | new 구현 상태 |
|------|------|----------------|
| **관리자(교육담당·HR)** | 조직 단위 현황·계획 | **구현됨**: 대시보드 개요·역량 분석(필터·KPI·부서별 차트·추이·벤치마킹). 나머지(임직원 테이블·인사이트·히트맵·전략·ROI·고성과자·리스크)는 **미구현** |
| **일반 사용자(임직원)** | 진단 응시자, 학습자 | **미구현**: 홈·KAPP 진단·마이 대시보드·교육 큐레이션·나의 성장·알림·설정 |

## 2. 사용자 여정과 데이터 흐름 (new 기준)

### 2.1 현재 new에서의 페이지/라우팅
- **`/`**: 로그인(자리표시) → 제출 시 `/dashboard`로 이동. (실제 인증 없음.)
- **`/admin`**: 관리자 대시보드 안내 문구만 표시. (로그인 후 접근 가정, 실제 기능 없음.)
- **`/dashboard`**: 관리자 대시보드 **개요** — KPI 카드, 부서별 역량/스킬 차트, 요약 블록, **역량 벤치마킹** 섹션.
- **`/dashboard/competency`**: **역량 분석** — 부서·기간 필터, 요약 KPI, 부서별 역량 막대 차트, 기간별 추이 차트, 탭(현황&비교 / 격차&리스크 / 전략&성과). 후두 탭은 “준비중” 플레이스홀더.

데이터는 **data/ 폴더 내 JSON·TS**에서 로드(프론트에서 직접 import). API 없음.

### 2.2 데이터가 쓰이는 방식 (new 현재)
- **관리자 대시보드**:  
  - 개요 KPI: 현재 **하드코딩** (1,234명, 12개 프로그램, 87%, 4.2 등).  
  - 부서별 역량 현황: **departmentSkillPerformance** (년/월 필터).  
  - 부서별 성과: **departmentPerformance** (기간 옵션·부서별 값).  
  - 벤치마킹: **organizationData.json** + **industryBenchmarks.json** → 부서·업계 선택 시 레이더·인사이트.
- **역량 분석**: **competencyRawData**(TS 생성) — 부서·날짜·역량 타입·점수. 필터(부서·기간)·일/월 추이·요약 KPI·막대 차트·추이 라인에 사용.

### 2.3 미구현 영역의 기획적 목적 (나중에 구현 시 참고)
- **홈**: 서비스 소개, 진단 유도.
- **KAPP 진단**: 정보 입력 → 문항 → 제출 → 결과 저장 후 마이 대시보드로 이동.
- **마이 대시보드**: 진단 결과 기반 DNA·KAPP 점수·레이더·시장 포지션·커리어 경로·IDP·게이미피케이션 등.
- **교육 큐레이션**: 맞춤 추천·전체 과정·필터·학습 경로.
- **나의 성장**: 1% 팁·AI 멘토·역량 인증서.
- **알림·설정**: 푸시 권한·알림/학습 설정.

## 3. 페이지별 기능·목적 (new 기준)

### 3.1 new에 실제로 존재하는 페이지
| 경로 | 목적 | 데이터 소스 |
|------|------|-------------|
| `/` | 로그인(자리표시) | 없음 |
| `/admin` | 관리자 진입 안내 | 없음 |
| `/dashboard` | 관리자 대시보드 개요 | departmentPerformance, departmentSkillPerformance, organizationData, industryBenchmarks |
| `/dashboard/competency` | 역량 분석(부서·기간·역량) | competencyRawData, organizationData, industryBenchmarks |

### 3.2 origin에는 있으나 new에는 아직 없는 페이지
| 기능/화면 | 기획적 목적 |
|-----------|-------------|
| 홈 | 랜딩·진단 CTA |
| KAPP 진단 | 정보 수집 + KAPP 4차원 측정 |
| 마이 대시보드 | 진단 결과 소비·성장·교육 유도 |
| 교육 큐레이션 | 추천·과정 탐색·학습 경로 |
| 나의 성장 | 1% 팁·AI 멘토·인증서 |
| 알림·설정 | 알림/학습 설정 |

### 3.3 구현 상태 요약
- **현재 new에는 관리자 대시보드(대시보드 개요·역량 분석)만 구현**되어 있고, **KAPP 진단·학습자 마이 대시보드·교육 큐레이션·나의 성장·알림 페이지는 미구현**이다.  
- 관리자 쪽에서도 **요약 KPI 카드 숫자**는 아직 JSON(summaryStats 등)이 아닌 하드코딩이며, **임직원 테이블·AI 인사이트·스킬 갭 히트맵·전략 매핑·ROI·고성과자·스킬 리스크** 등은 미구현이다.

---

## 4. 레이아웃·데이터 표시 정책 (new)

- **레이아웃**: 스크롤만 있는 형태가 아니라 **사이드바 + 헤더**(+ 필요 시 우측 패널) 구조. 대시보드 진입 후 모든 관리자 화면은 이 전제로 설계됨.
- **데이터 표시**: 관리자·학습자 모두 **간소화** 방향.  
  - **줄인 부분**: origin 대비 블록 수·차트 종류·테이블 컬럼을 줄이고, “요약 KPI + 부서 차트 + 벤치마킹 + 역량 분석” 등 핵심만 노출.  
  - **남긴 블록**: 대시보드 개요(KPI 카드, 부서별 역량/스킬 차트, 요약·다음 액션), 역량 분석(필터·KPI·부서별 막대·추이·벤치마킹·인사이트).

---

## 5. 기술 스택 (new)

- **현재 new 프로젝트는 Next.js(App Router) 기반**이며, Tailwind CSS·Noto Sans KR 등 사용.  
- HTML/CSS/JS 기반 origin의 인라인/목 데이터는 **data/ 폴더 JSON·TS**로 이전된 구조로 이해하면 됨.

---

## 6. 정리

- new는 **관리자 대시보드(개요·역량 분석)** 와 **사이드바·헤더 레이아웃**, **data/ JSON 더미**까지 반영된 상태이다.  
- **비전·목표·사용자 역할**은 origin과 동일하나, **구현된 페이지·데이터 흐름**은 위 3·4절을 기준으로 하며, 미구현 영역은 2.3·3.2에서 기획적 목적만 정리하였다.
