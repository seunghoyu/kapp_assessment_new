# KAPP 진단 데이터 안내 (비개발자용)

> KAPP 진단에서 **어떤 정보를 묻고**, **어떤 결과가 나오는지** 한눈에 보는 문서입니다.  
> 기획·운영·교육 담당자도 이해할 수 있도록 작성했습니다.

---

## 1. 진단 전에 입력하는 정보 (기본정보)

진단을 시작하기 전, 아래 정보를 한 번만 입력합니다.

| 구분 | 입력 항목 | 설명 | 필수 여부 |
|------|------------|------|-----------|
| **기본 정보** | 이름 | 진단 결과·리포트에 표시되는 이름 | 필수 |
| | 이메일 | 연락용 (선택) | 선택 |
| **산업·직무** | 산업군 | IT, 금융, 교육, 의료, 제조, 유통/리테일, 마케팅/광고, 호텔/관광, 법률/회계, 기타 중 선택 | 필수 |
| | 세부 직무 | 선택한 산업에 따라 달라지는 직무 목록에서 선택 (예: IT → 개발자(Backend), 개발자(Frontend) 등) | 필수 |
| **경력 정보** | 직급 | 인턴, 사원, 주임, 대리, 과장, 차장, 부장, 임원 중 선택 | 필수 |
| | 연차 | 1년 미만(신입) ~ 16년 이상 구간 중 선택 | 필수 |
| **조직 정보** | 회사명 | 소속 회사 이름 (선택) | 선택 |
| | 기업 규모 | 대기업(1000명 이상), 중견(300~999명), 중소(50~299명), 스타트업(50명 미만) 중 선택 | 필수 |
| **진단 목표** | 목표 | 승진 준비 / 이직·전직 준비 / 직무 스킬 향상 / 커리어 전환 — **복수 선택 가능** (체크리스트) | 선택 |

- 위 항목은 **어떤 문항이 나올지**(산업·직무·직급·연차)와 **이메일/워크플로우 시나리오**(산업)를 결정하는 데 사용됩니다.
- 진단 목표는 문항 선정에는 쓰이지 않고, **결과 해석·추천 교육** 등에만 활용됩니다.

---

### 1.1 산업군 → 세부 직무 선택 구조 (더미데이터 기준)

진단 시 **산업·직무**는 두 단계로 선택합니다.

1. **산업군**을 먼저 고릅니다.
2. 선택한 산업에 따라 **세부 직무** 목록이 정해지고, 그중 하나를 고릅니다.

즉, **선택한 산업 → 그다음 세부 직무** 순서이며, 세부 직무는 산업별로 완전히 다릅니다.  
아래는 `data/kappDiagnosis/userInfoOptions.json` 더미데이터 기준 **산업군별 세부 직무** 전체 목록입니다.

## 2. 진단이 진행되는 순서 (단계별)

1. **정보 입력** → 위 표의 항목 입력 후 시작
2. **지식 문항** → 난이도가 맞춤으로 바뀌는 퀴즈 (쉬움/보통/어려움)
3. **적용 문항** → 업무 시나리오를 읽고 선택하는 문항
4. **성과 문항** → KPI·성과 관련 선택 문항
5. **디지털 인바스켓(E-tray)** → 받은 이메일을 처리하는 시뮬레이션 (산업별로 다른 이메일 세트)
6. **AI 워크플로우** → 업무 상황에서 AI 도구를 어떻게 쓰는지 선택 (산업별 1개 시나리오)
7. **결과** → 영역별 점수·종합 점수·강점/개선 영역 표시

## 3. 진단 결과에 포함되는 것

진단을 끝내면 아래와 같은 **결과**가 만들어집니다.

| 구분 | 내용 |
|------|------|
| **입력했던 정보** | 이름, 산업·직무, 직급, 연차, 기업 규모, 진단 목표 등 (다시 보기·리포트용) |
| **영역별 점수** | 지식 / 적용 / 성과 / 생산성(인바스켓 등) 점수 |
| **종합 점수** | 위 네 영역을 합산한 전체 점수 |
| **지식 수준** | 지식 문항에서 적응형으로 측정된 레벨(1~5) |
| **문항별 답** | 어떤 문항에 어떤 답을 선택했는지 (상세 분석·리포트용) |
| **인바스켓 기록** | 어떤 이메일에 어떤 액션(열기/답장/전달/보관 등)을 했는지, 소요 시간 |
| **AI 워크플로우 기록** | 어떤 옵션을 선택했는지, 정답 여부 |

- 이 결과는 **한 번의 진단**을 한 덩어리로 저장할 수 있도록 정리된 형태로 보관됩니다.
- 실제 서비스에서는 이 결과를 **마이페이지·관리자 대시보드**에서 조회·집계할 수 있게 연동합니다.

## 4. 사용하는 데이터 파일 요약 (운영·기획 참고)

| 용도 | 설명 | 비고 |
|------|------|------|
| **정보 입력 옵션** | 산업·직무 목록, 직급, 연차, 기업 규모, 진단 목표 목록 | 선택지 추가·수정 시 이 데이터를 수정 |
| **진단 결과 샘플** | 결과가 어떤 항목으로 저장되는지 예시 | 리포트·대시보드 화면 설계 시 참고 |
| **진단 Raw 예시** | 정보 입력 + 단계별 문항/답 + 인바스켓·AI워크플로우 + 최종 결과를 한 건으로 표현한 예시 | 데이터 구조·연동 설계 시 참고 |

문항이 **어떤 경우에 어떻게 나오는지**는 **「KAPP 진단 문항 구성 로직」** 문서에서 경우별로 자세히 설명합니다.

---

## 5. 개발·연동 시 참고 (요약)

- **정보 입력 옵션**: `data/kappDiagnosis/userInfoOptions.json`
- **결과 예시(플랫)**: `data/kappDiagnosis/assessmentResultSample.json`
- **진단 Raw 1건(정규화)**: `data/kappDiagnosis/kappDiagnosisRawData.json`
- **문항 구성 로직(경우별)**: 문서 목록의 「KAPP 진단 문항 구성 로직」

관리자 대시보드와 연동할 때는, 진단 제출 시 위 결과를 서버에 저장하고, 대시보드에서는 기간·부서·산업별로 조회·집계하는 구조를 권장합니다.
